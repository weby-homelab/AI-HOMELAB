# 🤝 Гайд для Контриб'юторів | Contributing Guide

Дякуємо за бажання зробити свій внесок у **AI-HomeLab**! 🇺🇦

## 📋 Перед початком

1. Прочитайте [МЕМОРАНДУМ](./README.md#-меморандум-та-філософія-проєкту) — він визначає стек та принципи проєкту
2. Ознайомтесь з [SECURITY.md](./SECURITY.md) — безпека є пріоритетом
3. Перевірте [Issues](https://github.com/weby-homelab/AI-HOMELAB/issues) — можливо, ваша ідея вже обговорюється

## 🔄 Workflow

Ми використовуємо класичний **Fork → Branch → PR** workflow:

```
1. Fork репозиторій
2. Створіть Issue з описом (feature/bug/docs)
3. Створіть гілку: git checkout -b feature/назва-фічі
4. Зробіть зміни та протестуйте
5. Commit з підписом: git commit -S -m "feat: опис змін"
6. Push: git push origin feature/назва-фічі
7. Створіть Pull Request з посиланням на Issue
```

## 📁 Куди контриб'ютити

| Розділ | Що додавати |
|---|---|
| `/benchmarks` | Тести швидкості генерації (t/s), енергоефективності (t/s/W), оптимізації Speculative Decoding (MTP) та низькобітних квантів |
| `/configs` | Docker-compose конфіги, налаштування Ollama, vLLM, LiteLLM, Qdrant та готових MCP-серверів |
| `/templates` | Шаблони коду для LangGraph/CrewAI/PydanticAI, кастомні MCP-сервери, скрипти довготривалої пам'яті |
| `/projects` | Ваші пет-проєкти, OSINT-помічники та автоматизація розумного дому |
| `/docs` | Документація, туторіали, дослідження ландшафту та гайди енергоавтономності |

## ✅ Чеклист перед PR

- [ ] Код/конфіг працює на вашій машині
- [ ] Немає секретів (ключів, паролів, токенів) у коді
- [ ] Використано тільки дозволений стек (див. Меморандум)
- [ ] Додано README.md у директорію вашого контриб'юту
- [ ] PR має зрозумілий опис та посилання на Issue

## 🚫 Що НЕ приймається

- Будь-який софт чи моделі походженням з РФ (YandexGPT, GigaChat тощо — перебувають під **СУВОРОЮ ТА ДОВІЧНОЮ ЗАБОРОНОЮ**)
- Китайські моделі чи софт (DeepSeek, Qwen, Ornith) у ролі основних інфраструктурних джерел за замовчуванням (вони дозволені **ВИКЛЮЧНО** для локального тестування, порівняння та підвищення швидкості кодування згідно з Меморандумом)
- Конфігурації з хардкодженими секретами (паролі, токени, приватні GPG ключі)
- PR без опису або без зв'язаного Issue
- Код/конфіги без базової документації

## 💡 Ідеї для першого контриб'юту

- Додайте бенчмарк вашого заліза (GPU чи Apple Silicon) у папку `/benchmarks`
- Поділіться досвідом та конфігураціями оптимізації KV-cache або Speculative Decoding (MTP)
- Створіть та додайте корисний локальний MCP-сервер (наприклад, для роботи з локальними API)
- Поділіться досвідом роботи домашньої ШІ-лабораторії від інвертора чи EcoFlow під час блекаутів
