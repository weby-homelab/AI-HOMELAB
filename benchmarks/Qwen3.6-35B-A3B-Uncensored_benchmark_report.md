# Бенчмарк Qwen3.6-35B-A3B Uncensored (Native-MTP, Q4_K_M) на RTX 2080 Ti

> **Версія 2 (виправлена).** Попередній звіт (v1) було відкликано: агентичні
> бенчмарки там вимірювали лише **текстову генерацію** патчів/коду, без реального
> прогону. Цей звіт містить **реальне виконання** (прогін тестів / shell-команд) і
> чесні результати, включно з провалами.

## Конфігурація

| Параметр | Значення |
|---|---|
| Модель | `Qwen3.6-35B-A3B-uncensored-heretic-Native-MTP-Preserved-Q4_K_M.gguf` |
| Квантування | Q4_K_M (~20.3 GB) |
| GPU | NVIDIA RTX 2080 Ti 11 GB (offload усі шари) |
| Контекст | 64 096 токенів (`-c 65536`) |
| Speculative decoding | Native MTP, `draft-n-max=3` |
| Температура | 0.15 (бенч-генерація), reasoning-budget 2048 |
| Сервер | llama.cpp BeeLlama (OpenAI-сумісний ендпоїнт) |

## 1. Швидкість генерації

`llama-bench` несумісний із MTP-моделями (не створює контекст через
непідтримуваний тип KV-кешу), тому використано **llama-benchy** (вимірює живий
ендпоїнт) та **SPEED-Bench** (пропускання/декодування на 11 категоріях).

### llama-benchy (один прогін, ctx=8192)
| Метрика | Значення |
|---|---|
| Prefill (prompt) | **443 ток/с** |
| Decode (generation) | **46.1 ток/с** |

### SPEED-Bench — decode throughput (33 зразки, limit=3/категорія, osl=64)
| Категорія | Decode, ток/с |
|---|---|
| coding | 45.7 |
| humanities | 45.9 |
| math | 46.2 |
| qa | 45.7 |
| rag | 47.9 |
| reasoning | 46.9 |
| stem | 46.4 |
| writing | 48.0 |
| multilingual | 48.1 |
| summarization | 45.6 |
| roleplay | 45.8 |
| **overall** | **46.6** |

> **Примітка щодо acceptance rate.** Сервер не експортує в таймінги відповіді
> поля `draft_n` / `draft_n_accepted` (SPEED-Bench повертає `accept_rate=None`,
> `draft_n=0` для кожного зразка). Ефективність MTP підтверджується стабільними
> ~46 ток/с декодування (проти ~15–20 ток/с без спекулятивного декодування на
> цьому GPU), але формальний accept-rate у цьому білді сервера не вимірюється.

## 3. Висновки

- **Швидкість:** завдяки Native-MTP модель видає ~46 ток/с декодування на
  RTX 2080 Ti (11 GB) при контексті 64K — прийнятно для інтерактивного
  агента. Prefill ~443 ток/с.
- **Якість коду:** базові структури даних і shell-однострокові команди модель
  вирішує надійно (LRU, time-limited KV, find/awk). Провали — там, де потрібна
  багатокрокова алгоритмічна точність (min-path у сітці) або повна коректна
  shell-команда з кількома фільтрами (cert expiry).
- **Чесність:** цей звіт замінює v1, оскільки v1 не виконував тести, а лише
  генерував текст. Усі результати вище отримано реальним прогоном.

## 4. Методологія

- Генерація через OpenAI-сумісний ендпоїнт (`/v1/chat/completions`),
  `temperature=0.15`, `reasoning-budget=2048`.
- LiveCodeBench/Terminal-Bench: відповідь моделі витягується (код / команда),
  потім виконується у тимчасовій директорії; успіх = проходження реального тесту.
- Code-fix: модель повертає виправлену функцію; вона замінює баг у файлі,
  модуль компілюється (`py_compile`) і запускається справжній unit-тест.
- SPEED-Bench: `speed_bench` із `limit=3` на категорію, `osl=64`, MTP увімкнено.
