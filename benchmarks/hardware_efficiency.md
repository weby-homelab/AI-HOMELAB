# ⚡ Hardware Efficiency Benchmarks: Енергоефективність AI-інференсу (Червень 2026)

> **Мета:** Об'єктивне порівняння апаратного забезпечення для локального AI-інференсу в умовах обмеженого бюджету та нестабільного енергопостачання (блекаути). Головна метрика — **tokens/second на ватт (t/s/W)**.

> [!NOTE]
> Бенчмарки проводяться на **квантованих моделях** (Q4_K_M або сумісних), які є стандартом для домашніх лабораторій. Результати для FP16 моделей будуть суттєво відрізнятися.

## 📏 Методологія тестування

### Інструменти

| Інструмент | Призначення | Команда |
|---|---|---|
| **Ollama** | Інференс LLM (SLM) | `ollama run <model>` |
| **llama-server (llama.cpp)** | Професійний API інференс | `llama-server -m model.gguf` |
| **nvidia-smi** | Моніторинг GPU (TDP/VRAM) | `nvidia-smi dmon -s pucvmet -d 1` |
| **powermetrics** | Споживання на macOS | `sudo powermetrics --samplers gpu_power` |
| **HWMonitor / sensors** | Системний моніторинг | `sensors` |

### Метрики

| Метрика | Позначення | Опис |
|---|---|---|
| **Prompt eval** | `pp` (prompt processing) | Швидкість обробки вхідного контексту (tokens/s) |
| **Token generation** | `tg` (token generation) | Швидкість генерації нових токенів (tokens/s) |
| **Time to First Token** | `TTFT` | Час від запиту до першого згенерованого токена (мс) |
| **Total Power Draw** | `TDP_actual` | Фактичне споживання системи під навантаженням (W) |
| **Energy Efficiency** | `t/s/W` | Токенів за секунду на кожен ватт споживання |
| **Cold Start** | `CS` | Час від запуску до першої відповіді (с) |

### Стандартний тест

Для вимірювання швидкості використовується стандартизований промпт:
```bash
PROMPT="Розкажи детально про архітектуру Transformer та механізм self-attention. Поясни різницю між encoder та decoder. Наведи приклади застосування."
```

## 🧊 Аналіз Cold Start Latency

### Виміри Cold Start (Завантаження моделі в пам'ять)

| Конфігурація | Диск | Cold Start (12B LLM) | Cold Start (RAG DB) | Примітка |
|---|---|---|---|---|
| RTX 3060 + NVMe SSD | NVMe Gen3 | ~3.5s | ~8s | Стандарт |
| RTX 3060 + SATA SSD | SATA III | ~6s | ~14s | 2x повільніше |
| M4 Pro + Unified Memory | Internal NVMe | ~1.5s | ~4s | Миттєво |
| RTX 3060 + HDD | HDD 7200 | ~18s | ~35s | Неприйнятно |
| RTX 3060 + Swap | NVMe + Swap | ~25s | ~45s | ⚠️ VRAM overflow |

### Зниження Cold Start на серверах:
1. Використовуйте параметр keep-alive, щоб запобігти вивантаженню моделей:
   ```bash
   export OLLAMA_KEEP_ALIVE=24h
   ```
2. Зберігайте моделі на швидких NVMe накопичувачах (швидкість зчитування від 3500 MB/s).

## 🔀 VRAM Contention: LLM + Embedding

При запуску RAG-пайплайнів у VRAM завантажуються дві моделі: LLM (напр. Gemma 4 12B, ~8.2 GB) та Embedding-модель (напр. nomic-embed-text, ~300 MB).

```
Бюджет VRAM (RTX 3060 12GB):
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
│ System Overhead │  Embedding  │     Gemma 4 12B     │ Free │
│     ~0.6 GB     │   ~0.3 GB   │       ~8.2 GB       │2.9 GB│
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✅ Робота в межах 12GB VRAM — стабільна (залишається буфер для KV-кешу).
```

### Рішення для оптимізації VRAM:
* **CPU Offloading:** Запуск важких Embedding-моделей (напр. BGE-M3) на CPU.
* **Lightweight Embeddings:** Використання легких моделей, таких як `all-minilm` (80 MB VRAM).

## 🎯 Рекомендації по тієрах

### Тієр 1: Блекаут-пріоритет (Автономність від EcoFlow)
* **Апаратна частина:** Mac Mini M4 16GB / MacBook Air M4 (TDP ~22W).
* **Моделі:** Gemma 4 12B (Q3_K_S) + all-minilm.
* **Швидкість:** ~12 токенів/сек.
* **Автономність:** до **45 годин** роботи від EcoFlow 1024Wh.

### Тієр 2: Оптимальний баланс (Розробник)
* **Апаратна частина:** RTX 3060 12GB (обмежена до PL 90W) + CPU Xeon/Core i5.
* **Моделі:** Phi-4 14B (Q4_K_M) або Gemma 4 12B (Q4_K_M).
* **Швидкість:** ~23-26 токенів/сек.
* **Автономність:** до **7-9 годин** роботи від EcoFlow 1024Wh.

### Тієр 3: Професійна лабораторія
* **Апаратна частина:** M4 Max (48GB / 64GB) або RTX 4060 Ti 16GB.
* **Моделі:** Gemma 4 26B MoE або Ornith 1.0 35B (з MTP).
* **Швидкість:** ~25-46 токенів/сек.
* **Автономність:** до **25 годин** (на M4 Max) від EcoFlow 1024Wh.

## 🔋 Оптимізація енергоспоживання при роботі від інвертора / повербанка

Для досягнення максимальної автономності вашої локальної лабораторії під час блекаутів використовуйте такі налаштування:

1. **Миттєве вивантаження з пам'яті (Standby Tax):** Звичайний режим очікування моделі у VRAM споживає 15-30W на GPU. Встановіть мінімальний час життя моделі в пам'яті, щоб GPU одразу переходив у глибокий режим енергозбереження:
   ```bash
   export OLLAMA_KEEP_ALIVE=10s
   ```
2. **Обмеження апаратного ліміту (Nvidia GPU):** Зрізайте пікові стрибки споживання без значного падіння швидкості генерації:
   ```bash
   # Обмеження RTX 3060 до 80W (за замовчуванням 170W)
   sudo nvidia-smi -pl 80
   # Обмеження RTX 4060 Ti до 90W (за замовчуванням 165W)
   sudo nvidia-smi -pl 90
   ```
3. **Обмеження паралельності:** Використовуйте `OLLAMA_NUM_PARALLEL=1`, щоб уникнути одночасного розігріву кількох ядер процесора/відеокарти.

## 🔬 Як провести власні бенчмарки

### Скрипт автоматизації вимірювання швидкості
Створіть скрипт `benchmark.sh`:
```bash
#!/usr/bin/env bash
set -euo pipefail
MODEL="${1:?Usage: $0 <model_name>}"
PROMPT="Розкажи детально про архітектуру Transformer та механізм self-attention."
ollama stop "${MODEL}" 2>/dev/null || true
sleep 2
echo "🧪 Вимірювання швидкості для ${MODEL}..."
ollama run "${MODEL}" "${PROMPT}" --verbose 2>&1 | grep -E "eval rate|total duration"
```

### Моніторинг фактичного споживання (NVIDIA)
```bash
nvidia-smi dmon -s p -d 1 | awk '{print strftime("%H:%M:%S"), $2"W"}'
```
