# 🔌 Unified Agent Infrastructure Stack

Цей інфраструктурний шаблон призначений для швидкого розгортання повноцінного мультиагентного середовища у вашій домашній лабораторії. Замість окремого запуску кожного сервісу, цей Docker-compose конфіг об'єднує інференс, проксі-роутинг, векторне сховище, візуальну автоматизацію та веб-інтерфейс в єдиний стек.

---

## 🏗️ Склад стеку

1. **Ollama (`agent-ollama`)**: Двигун локального виконання LLM та Embedding моделей.
2. **Qdrant (`agent-qdrant`)**: Високопродуктивна векторна база даних для збереження знань (RAG) та довготривалої пам'яті агентів.
3. **LiteLLM (`agent-litellm`)**: Роутер та OpenAI-сумісний проксі-сервер. Дозволяє стандартизувати виклики моделей та легко налаштовувати fallbacks (переключення на резервні хмарні API у разі відключення GPU або блекауту).
4. **n8n (`agent-n8n`)**: Потужна платформа для візуальної оркестрації агентів та інтеграції їх із зовнішніми сервісами (Telegram, Gmail, Notion тощо).
5. **Open WebUI (`agent-open-webui`)**: Веб-чат інтерфейс для користувачів та панель керування агентами.

---

## ⏱️ Швидкий запуск

### 1. Підготовка конфігурації
Переконайтеся, що файли `docker-compose.yml` та `litellm-config.yaml` знаходяться в одній директорії.

### 2. Увімкнення GPU-прискорення (Опціонально)
Якщо на вашому сервері встановлено графічну карту Nvidia та `nvidia-container-toolkit`, відкрийте `docker-compose.yml` та розкоментуйте наступні рядки у сервісі `ollama`:

```yaml
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: all
              capabilities: [gpu]
```

### 3. Запуск сервісів
Виконайте команду у терміналі:

```bash
docker compose up -d
```

Це завантажить необхідні образи та запустить усі контейнери у фоновому режимі.

Після запуску ви можете отримати доступ до інтерфейсів за наступними портами:

| Сервіс | Порт хоста | Адреса | Призначення |
|---|---|---|---|
| **Open WebUI** | `3080` | `http://localhost:3080` | Чат інтерфейс користувача |

## ⚙️ Налаштування оркестрації в n8n

Для того, щоб підключити n8n до вашого локального AI-стеку:

1. Відкрийте n8n за адресою `http://localhost:5678` та пройдіть первинне налаштування.
2. Створіть новий Workflow та додайте блок **Advanced AI** -> **OpenAI Chat Model**.
3. У налаштуваннях з'єднання (Credentials) вкажіть:
   * **Resource:** `OpenAI`
   * **API Key:** `sk-dummy-key` (будь-який рядок, оскільки LiteLLM локальний)
   * **Base URL:** `http://agent-litellm:4000/v1`
4. У параметрі **Model** впишіть назву моделі з конфігурації LiteLLM, наприклад `phi-4` або `gemma-3-4b`.
5. Для підключення RAG використовуйте блок **Qdrant Vector Store** із підключенням до `http://agent-qdrant:6333`.
