### Детальний огляд інструментів

#### 1. 💻 Cursor
**Cursor** — це сучасний AI-орієнтований редактор коду, побудований на базі VS Code.
- **Можливості:** Інтелектуальний автокомплит (Cursor Tab), чат з контекстом всього репозиторію (`@Workspace`), автоматичний рефакторинг та швидке виправлення помилок лінтера.
- **Використання:** Ідеально підходить для швидкого розгортання та написання скриптів без постійного перемикання в браузер.

#### 2. 🧪 Google AI Studio
**Google AI Studio** — професійне середовище для розробників, що надає прямий доступ до моделей сімейства Gemini (включаючи Gemini 1.5 Pro та Gemini 1.5 Flash з гігантським контекстним вікном до 2 мільйонів токенів).
- **Можливості:** Створення системних інструкцій (System Prompts), налаштування параметрів температури, тестування структурованого виводу (JSON Schema) та безкоштовні квоти на API-запити.
- **Використання:** Незамінний інструмент для швидкого тестування логіки та аналізу великих масивів документів.

#### 3. 🎨 Canva Magic Studio
Набір інтелектуальних функцій всередині популярного графічного редактора **Canva**.
- **Можливості:** Видалення фону в один клік (Background Remover), інструмент "Magic Expand" для розширення меж зображень, текстовий генератор зображень та інтегрований монтаж відео за допомогою AI.
- **Використання:** Швидке створення банерів, ілюстрацій для документації та промо-матеріалів.

#### 4. ⚡ Krea.ai
Платформа для генерації зображень у реальному часі.
- **Можливості:** Real-time Generation дозволяє бачити зміни на зображенні прямо під час малювання ескізу або руху курсора. Також містить інструменти для апскейлу (Enhance) до 4K без втрати стилю.
- **Використання:** Швидке концепт-артування, генерація текстур та покращення якості ілюстрацій.

#### 5. 🎬 Higgsfield / getimg.ai
Потужні AI-агрегатори для роботи з відео та складною графікою.
- **Можливості:** Об'єднують доступ до топових моделей генерації відео, таких як Kling, Google Veo, та Runway Gen-3.
- **Використання:** Генерація коротких відеороликів, анімація статичних зображень та перетворення тексту у відео без необхідності купувати дорогі підписки на кожен сервіс окремо.

#### 6. 🎙️ ElevenLabs
Лідер у сфері синтезу мовлення (Text-to-Speech) та голосового клонування.
- **Можливості:** Створення надреалістичного озвучення текстів різними мовами (включаючи якісну українську), клонування власного голосу за коротким аудіозаписом, генерація звукових ефектів (Sound Effects).
- **Використання:** Озвучення подкастів, створення голосових сповіщень для розумного дому, локальна генерація аудіо-контенту.

#### 7. 🔗 n8n (Self-hosted)
Безкоштовна self-hosted альтернатива Zapier з відкритим кодом.
- **Можливості:** Візуальне створення ланцюжків автоматизації. Має нативні ноди для AI: `Advanced AI`, де можна підключати агенти, ланцюжки (LangChain), бази векторних ембедінгів та локальні LLM.
- **Використання:** Створення автономних Telegram-ботів, парсинг новин, автоматичне оновлення звітів та синхронізація баз даних без передачі конфіденційних даних стороннім хмарам.

#### 8. 📓 Obsidian + AI
Локальний текстовий редактор Markdown-нотаток, перетворений на персональний AI-хаб.
- **Можливості:** Використання плагіна *Smart Connections* для створення локального RAG (Retrieval-Augmented Generation) на основі ваших записів. AI аналізує зв'язки між файлами та допомагає шукати інформацію.
- **Використання:** Ведення "Другого мозку" (Second Brain), розумний пошук по локальних базах знань без виходу в інтернет.

## 💡 7 AI-Лайфхаків для Максимальної Продуктивності

> [!IMPORTANT]
> Використання правильних методологій промптингу та архітектури взаємодії з моделями дозволяє підвищити якість відповідей AI на 40-60%.

### 1. Ланцюжок перевірки (Chain-of-Verification / CoVe)
Метод боротьби з галюцинаціями LLM. Замість того, щоб вірити першій відповіді моделі, ви змушуєте її критично оцінити власні висновки.
- **Крок 1:** Запит до моделі (отримання початкової відповіді).
- **Крок 2:** Промпт: *"Склади список фактів, які ти використав у попередній відповіді, та перевір кожен з них на точність за допомогою незалежних логічних кроків. Створи таблицю перевірки."*
- **Крок 3:** Фінальне коригування на основі знайдених помилок.

### 2. Персональний RAG через NotebookLM
Сервіс від Google, який дозволяє завантажити до 50 джерел (PDF, Markdown, посилання, нотатки) і створює ізольовану базу знань.
- **Лайфхак:** NotebookLM не використовує ваші документи для загального навчання моделей. Ви отримуєте відповіді суворо в межах контексту завантажених файлів з точними цитатами та посиланнями на першоджерела. Ідеально для аналізу складної документації чи книг.

### 3. Паралельний промптинг (Parallel Prompting)
Одночасне тестування одного і того ж запиту на різних архітектурах моделей для знаходження золотого стандарту.
- **Реалізація:** Наприклад, відправка запиту в Claude 3.5 Sonnet (для логіки та програмування), Gemini 1.5 Pro (для аналізу величезного контексту) та GPT-4o (для структурування). Це дозволяє обрати найкраще формулювання або скомбінувати їхні сильні сторони.

### 4. Голосовий робочий процес (Voice-first workflow)
Метод швидкої фіксації думок без використання клавіатури.
- **Схема процесу:** 
  $$\text{Голосове надиктування (Whisper/Live Dictation)} \rightarrow \text{LLM-структурування (Markdown)} \rightarrow \text{Синтез голосу (ElevenLabs)}$$
- **Опис:** Ви надиктовуєте хаотичний потік думок на телефон. Локальна чи хмарна модель перетворює це на структурований Markdown-документ (план або статтю). Потім ElevenLabs може озвучити фінальний результат вашим власним голосом для перевірки сприйняття на слух.

### 5. Локальний штучний інтелект (Strict Project Philosophy Compliance)

> [!WARNING]
> **Принцип 1: Технологічна гігієна (Technological Hygiene)**
> У межах екосистеми AI-HomeLab діє жорстке правило безпеки даних: ми категорично уникаємо використання моделей китайського походження (таких як Qwen або DeepSeek) через ризики неконтрольованого витоку телеметрії та даних користувачів.

Для локальних інференс-движків (Ollama, vLLM, Llama.cpp) дозволено використовувати виключно перевірені західні open-source та open-weights моделі:
1. **Meta LLaMA 4** (генеральні задачі, кодування, логічні ланцюжки)
2. **Mistral / Mixtral** (ефективні європейські моделі з відмінною підтримкою мов)
3. **Google Gemma 3 / Gemma 4** (легковагові та швидкі моделі для RAG)
4. **Microsoft Phi-4** (компактна модель для специфічних задач та обмежених ресурсів)

### 6. Петля автокритики (Auto-Critique Loop)
Змушує модель виправляти власні помилки програмування чи стилістики до того, як ви скопіюєте код.
- **Приклад промпту:**
  ```text
  [Твій початковий запит]
  ---
  Після написання коду, виконай роль суворого системного архітектора та аудитора безпеки. 
  Знайди 3 потенційні слабкі місця у своєму рішенні (продуктивність, безпека, обробка помилок). 
  Напиши критику, а потім надай покращену версію коду.
  ```

### 7. Делегація AI-to-AI (AI-to-AI Delegation)
Ланцюжки промптів, де вихідні дані однієї моделі стають інструкцією для іншої.
- **Схема взаємодії:**
  1. **GPT-4o (Drafting):** Створює первинну концепцію або детальний промпт для задачі.
  2. **Claude 3.5 Sonnet (Implementation):** Виконує основну роботу (пише чистий код або текст) на основі отриманого промпту.
  3. **Gemini 1.5 Pro (Review):** Аналізує фінальний результат, перевіряє його за допомогою гігантського контексту знань та вносить фінальні правки.
