# 🔋 БЛЕКАУТ-ГАЙД: Налаштування ШІ-лабораторії для роботи від EcoFlow

Цей гайд призначений для українських розробників та ентузіастів, які будують локальні AI-лабораторії в умовах нестабільного енергопостачання. Тут описано, як мінімізувати споживання енергії вашим залізом під час роботи від акумуляторів чи зарядних станцій (EcoFlow, Bluetti тощо).

---

## 🔌 1. Апаратна енергоефективність (Hardware Choice)

Вибір заліза визначає, скільки годин пропрацює ваша лаба під час відключення світла.

| Платформа | Робоче споживання | Споживання під навантаженням (LLM) | Оцінка автономності (на EcoFlow River 2, 256Wh) |
|---|---|---|---|
| **Apple Silicon (Mac mini M2/M4)** | 4–7 W | **15–25 W** | **~10-12 годин чату** |
| **Міні-ПК (Ryzen 7 7840HS)** | 8–12 W | **35–55 W** | **~4-5 годин чату** |
| **PC з RTX 3060 12GB (x86)** | 60–80 W | **180–220 W** | **~1 година чату** |
| **PC з RTX 4060 Ti 16GB** | 50–70 W | **140–170 W** | **~1.5 години чату** |

> [!TIP]
> **Apple Silicon** є абсолютним переможцем для блекаутів. Оскільки оперативна пам'ять є уніфікованою, моделі до 32B параметрів виконуються безпосередньо на NPU/GPU з мінімальним виділенням тепла та споживанням енергії.

## 🏎️ 2. Оптимізація x86 систем з GPU (Nvidia RTX)

Якщо ви використовуєте класичний ПК з дискретною відеокартою Nvidia, під час роботи від батареї обов'язково оптимізуйте ліміти живлення.

### Обмеження споживання GPU (Power Limit)
Сучасні відеокарти Nvidia мають дуже гнучкі ліміти живлення. Обмеження ліміту на 40-50% знижує продуктивність всього на 10-15%, але колосально економить заряд EcoFlow.

**Linux (потрібні права root):**
```bash
# Перевірити поточні ліміти та споживання
nvidia-smi -q -d POWER

# Встановити максимальний Power Limit (наприклад, 100W для RTX 3060, яка зазвичай споживає 170W)
sudo nvidia-smi -pl 100
```

**Windows:**
Використовуйте утиліту **MSI Afterburner** або офіційний `nvidia-smi` через командний рядок (CMD від імені адміністратора):
```cmd
nvidia-smi -pl 100
```

### Зниження частоти та напруги (Undervolting)
Зниження робочої напруги GPU через криву частот (Curve Editor в MSI Afterburner) дозволяє зекономити ще 20–30W без будь-якої втрати швидкості генерації токенів (t/s).

## 💻 3. Оптимізація систем без GPU (CPU-only)

Якщо моделі запускаються на процесорі (через CPU-інференс Ollama / llama.cpp):

### Обмеження кількості потоків (Threads Limit)
Запуск інференсу на 100% ядер процесора викликає експоненційне зростання споживання енергії (TDP процесора зростає з 15W до 65W+). 
Оптимальна кількість потоків для `llama.cpp` дорівнює кількості **фізичних P-ядер** (performance cores) вашого процесора, або половині логічних потоків.

**Конфігурація Ollama через змінну середовища:**
Встановіть кількість потоків (наприклад, 4 потоки для стабільної роботи):
```bash
export OLLAMA_NUM_PARALLEL=1
# Обмеження потоків для внутрішнього двигуна llama.cpp (зазвичай керується динамічно, але можна зафіксувати через конфіг)
```

## 🐳 4. Налаштування Docker та ОС під час блекаутів

### Вимкнення фонових сервісів
Зупиніть усі важкі контейнери, які не потрібні для роботи ШІ-моделей (наприклад, локальні бази даних, моніторинг Grafana/Prometheus):
```bash
docker stop $(docker ps -q | grep -v open-webui | grep -v ollama)
```

### Налаштування Power Governor (Linux)
Переведіть процесор у режим енергозбереження:
```bash
# Для процесорів Intel/AMD
sudo cpupower frequency-set -g powersave
```

## 🔋 5. Рекомендовані сценарії роботи від станцій

1. **Сценарій "Максимум автономності" (до 15 годин)**:
   * Хост: Mac Mini M2/M4 (8W середнє споживання).
   * Модель: `gemma3:4b` або `llama3.1:8b`.
   * Запуск безпосередньо через нативний клієнт Ollama без Docker (Docker споживає додаткові 2-3W).
2. **Сценарій "Компроміс" (до 4 годин)**:
   * Хост: Ноутбук з RTX 4060 Laptop GPU.
   * Power Limit GPU встановлено на **45W**.
   * Екран ноутбука вимкнено або яскравість на мінімумі, підключення через SSH/браузер з іншого пристрою (наприклад, планшета).
