# 🏗️ AI-HomeLab Reference Architectures: Еталонні архітектури локального ШІ

Цей документ представляє серію еталонних архітектур для побудови домашніх AI-лабораторій різного рівня. Вони орієнтовані на максимальну економічність, автономність та відповідність принципам безпеки проєкту AI-HomeLab.

> [!IMPORTANT]
> Відповідно до меморандуму проєкту, всі архітектурні рішення орієнтовані **виключно** на використання безпечних західних open-source моделей (Meta LLaMA, Google Gemma, Mistral, Microsoft Phi). Використання моделей з РФ та КНР (DeepSeek, Qwen) не тестується та не підтримується.

## ⏱️ Tier 1: Бюджетний локальний ШІ (Mini PC) — $300-500

### 1. Концепція
Цей тієр розрахований на новачків та користувачів із жорстким бюджетом або високим пріоритетом енергоефективності. Замість шумного та важкого ПК використовується ультракомпактний неттоп (Mini PC) на базі сучасних процесорів Ryzen з інтегрованими NPU (Ryzen AI з архітектурою XDNA/XDNA2) та потужною вбудованою графікою Radeon 780M/890M або Intel Ultra Gen 1/2.

### 2. Специфікація обладнання (BOM)
* **Апаратна платформа:** Mini PC (наприклад, Beelink SER7/SER8, Minisforum UM780 XTX, або Geekom A8).
* **Процесор:** AMD Ryzen 7 7840HS / Ryzen 7 8840HS / Ryzen 7 9840HS (8 ядер/16 потоків, інтегрований NPU 16-50 TOPS).
* **ОЗП:** 32 GB DDR5-5600 (SO-DIMM).
* **Накопичувач:** 1 TB NVMe SSD Gen 4 (швидкість читання >5000 MB/s для швидкого Cold Start).
* **Живлення:** Зовнішній блок живлення (GaN) на 65-100W.

### 3. Програмний стек
* **ОС:** Linux (Ubuntu 24.04 LTS Desktop/Server).
* **Шар інференсу:** **Ollama** (інференс на CPU + часткове прискорення через ROCm для Radeon 780M/890M, або чистий CPU-інференс із фіксованою кількістю потоків).
* **Інтерфейс:** **Open WebUI** (запуск у Docker-контейнері). Підтримка direct speech-to-speech для нативного аудіо.
* **База даних:** ChromaDB (локальна, in-memory) для RAG.

### 4. Рекомендований вибір моделей
* **Основна LLM:** `gemma4:e4b` (Q4_K_M, нативне аудіо + Vision) або `phi4:mini` (Q4_K_M) / `llama3.2:3b` (Q4_K_M).
* **Embedding модель:** `nomic-embed-text` (виконується на CPU / GPU).

### 5. Метрики та Енергоефективність
* **Швидкість генерації:** 25–35 t/s на моделях до 4B параметрів.
* **Споживання:** 8W у режимі спокою, 25-35W під навантаженням.
* **Блекаут:** Станція EcoFlow River 2 (256Wh) забезпечить **8-10 годин** роботи при лімітуванні `OLLAMA_KEEP_ALIVE=1m` (дозволяє GPU/CPU переходити в режим сну між запитами).

## 🤖 Tier 2: Просунутий ШІ & Агентна Інфраструктура (Nvidia GPU) — $1000-1500

### 1. Концепція
Базовий стандарт для розробників штучного інтелекту, що дозволяє запускати повноцінні мультиагентні системи, розробляти RAG-пайплайни та підтримувати локальних асистентів у реальному часі. Основним обчислювальним рушієм є дискретна відеокарта Nvidia з архітектурою Ada Lovelace, Ampere або Blackwell для стабільної роботи CUDA.

### 2. Специфікація обладнання (BOM)
* **Процесор:** Intel Core i5-13400 / Ryzen 5 7600 (6-8 ядер з високою однопоточною швидкістю).
* **Материнська плата:** Чипсет B760 / B650 з підтримкою PCI-e Gen 4/5.
* **Відеокарта:** **Nvidia RTX 3060 12GB GDDR6** (бюджетний вхід у 12GB VRAM), **RTX 4060 Ti 16GB GDDR6** (найкращий баланс ціни та VRAM) або **RTX 5070 12GB GDDR7** (максимальний bandwidth пам'яті 672 GB/s для надшвидкого інференсу).
* **ОЗП:** 64 GB DDR5-5600 (двоканальний режим).
* **Накопичувач:** 2 TB NVMe SSD Gen 4 (читання >7000 MB/s).
* **Блок живлення:** 650W-750W з сертифікатом 80 Plus Gold.

### 3. Програмний стек
* **ОС:** Linux (Ubuntu 24.04 LTS / Debian 12).
* **Оркестрація:** **LiteLLM** (як єдиний OpenAI-сумісний роутер та проксі-сервер) + **n8n** / **LangGraph** / **CrewAI** (для створення візуальних та кодових агентних робочих процесів з підтримкою Model Context Protocol (MCP)).
* **Шар інференсу:** **Ollama** з CUDA-прискоренням.
* **Векторна БД:** **Qdrant** (Docker-контейнер з NVMe-оптимізованим сховищем).
* **Інтерфейс:** **Open WebUI**, підключений до LiteLLM.

### 4. Рекомендований вибір моделей
* **Основна LLM (Reasoning/Coding):** `phi4:14b` (Q4_K_M, Reasoning) або `gemma4:12b` (Q8_0 / Q4_K_M).
* **Embedding модель:** `nomic-embed-text` (виконується на GPU).

### 5. Метрики та Енергоефективність
* **Швидкість генерації:** 25–45 t/s на моделях 12B-14B параметрів (швидкість суттєво вища на картах з шиною GDDR7).
* **Споживання:** 60W у режимі спокою, 120-200W під навантаженням (з обов'язковим обмеженням живлення GPU через `nvidia-smi -pl`).
* **Блекаут:** Станція EcoFlow Delta 2 (1024Wh) забезпечить **5-7 годин** роботи в режимі активної генерації (при обмеженні TDP відеокарти до 80-100W).

## ⚡ Tier 3: Максимальна якість & Розподілений інференс (Multi-GPU) — $3000+

### 1. Концепція
Професійне робоче місце для командної розробки, запуску великих моделей (до 70B+ параметрів), хостингу API для невеликих команд та проведення дрібних експериментів із fine-tuning (LoRA). Архітектура будується на використанні кількох графічних процесорів (Multi-GPU) з агресивним розподілом контексту, або на базі флагманського Apple Silicon (Mac Studio) як безшумного та енергоефективного рішення.

### 2. Специфікація обладнання (BOM)
* **Процесор:** AMD Ryzen 9 9900X / Ryzen 9 9950X або Intel Core Ultra 9 285K.
* **Материнська плата:** X870E / Z890 з підтримкою кількох слотів PCIe x16 (працюючих щонайменше у режимі x8/x8/x4).
* **Відеокарти:** **2x Nvidia RTX 3090 24GB GDDR6X** (вживані, сумарно 48GB VRAM), **2x RTX 4090 24GB** (максимальна швидкість) або **2x RTX 5090 32GB GDDR7** (ультра-преміум рішення, сумарно 64GB VRAM).
* **Альтернатива (Енергоефективність):** **Mac Studio M4 Ultra 192GB Unified Memory** (пропускна здатність 546 GB/s, запускає гігантські моделі до 120B+ параметрів на швидкості ~15-20 t/s при споживанні 40W).
* **ОЗП:** 128-256 GB DDR5 (із підтримкою ECC за можливості).
* **Накопичувач:** RAID 0 з 2x 2 TB NVMe SSD Gen 4/5 (читання >10000 MB/s для миттєвого завантаження моделей).
* **Блок живлення:** 1200W-1600W Titanium/Platinum клас.

### 3. Програмний стек
* **ОС:** Proxmox VE 8+ (як гіпервізор для поділу ресурсів) або чистий Ubuntu Server 24.04 із Docker.
* **Шар інференсу:** **vLLM** (використовує PagedAttention, Model Runner V2, тензорний паралелізм та Multi-Token Prediction з моделями-драфтерами) або **TensorRT-LLM**.
* **Роутинг та Лог-аналітика:** **LiteLLM** (з розподілом навантаження, підтримкою Prompt Caching та failover-роутингу) + **Langfuse** (для трейсингу та аудиту витрат API).
* **Інтеграція інструментів (MCP):** Встановлення локальних Python-основаних MCP-серверів для прямої взаємодії з Obsidian сховищем нотаток та веб-пошуком Odysseus.
* **Workflow:** n8n / LangGraph / CrewAI.
* **Векторна БД:** **Qdrant** (кластерний варіант з оптимізацією під NVMe).

### 4. Рекомендований вибір моделей
* **Основна LLM:** `llama3.3:70b` (Q4_K_M або Q5_K_M) або `gemma4:26b` (MoE) / `gemma4:31b` (Dense). Для систем з 64GB+ VRAM/RAM: `llama4:scout` (109B MoE, Q3/IQ3_XS).
* **Допоміжні LLM (агенти):** `phi4:14b` (Reasoning) або `gemma4:12b`.
* **Embedding модель:** `bge-m3` або `nomic-embed-text`.

### 5. Метрики та Енергоефективність
* **Швидкість генерації:** 20–30 t/s на моделях 70B при паралельних запитах; при увімкненому Speculative Decoding (MTP) швидкість на картах Nvidia досягає 45-60 t/s.
* **Споживання:** 120W у режимі спокою, 500-900W під навантаженням (або 35-50W для Mac Studio).
* **Блекаут:** Для GPU-конфігурацій потребує потужних інверторів (від 3 кВт) та акумуляторних збірок LiFePO4 (від 5 кВт·год). Конфігурація на Mac Studio M4 Ultra 192GB може безперервно працювати від станції EcoFlow Delta 2 Max (2048Wh) протягом **35-40 годин**.

## 🔌 Рекомендації щодо вибору заліза

1. **VRAM — це король:** Продуктивність ШІ-моделей залежить від швидкості та об'єму відеопам'яті. Завжди купуйте відеокарту з найбільшим об'ємом VRAM, який ви можете собі дозволити. 12GB VRAM — це абсолютний мінімум у 2026 році.
2. **Apple Silicon як альтернатива (ідеал для блекаутів):** Якщо вашим пріоритетом є безшумність, мінімальний TDP (15-45W) та робота під час тривалих відключень світла — Mac Mini / Studio на чіпах M3/M4 Pro/Max є неперевершеними. Завдяки уніфікованій пам'яті (до 192GB) вони запускають величезні моделі. Нове покоління чіпів M4 Pro (273 GB/s) та M4 Max (546 GB/s) суттєво звузило розрив у швидкості з дискретними відеокартами завдяки значному приросту пропускної здатності пам'яті.
3. **Швидкість SSD:** Не економте на SSD. Квантовані моделі об'ємом 10-40 GB повинні завантажуватися швидко. NVMe SSD Gen 4/5 дозволяє зменшити час холодного старту великих моделей до 1-3 секунд.
4. **Зниження порогу VRAM через 3-бітну квантизацію:** Сучасні алгоритми квантизації (такі як `IQ3_XS` або `Q3_K_L`) дозволяють запускати моделі розміром 70B параметрів на одній відеокарті з 24GB VRAM (наприклад, RTX 3090/4090) з мінімальною втратою точності, що раніше вимагало обов'язкового Multi-GPU конфігу.
5. **Speculative Decoding та Multi-Token Prediction (MTP):** При виборі програмного стеку обов'язково налаштовуйте MTP (доступно у `llama.cpp` та `vLLM`). Використання невеликої моделі-драфтера (наприклад, Gemma 4 E2B для Gemma 4 26B) збільшує швидкість генерації локально на 50–80% без зміни апаратного забезпечення.
