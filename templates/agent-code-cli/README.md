# 🤖 agent-code-cli: Шаблон ШІ-Агента для Кодування (Claude Code Style)

Цей модуль є навчальним та практичним шаблоном для побудови власного консольного ШІ-агента для роботи з кодом (у стилі Claude Code). Він демонструє проектування **Model Harness** (середовища виконання моделі): управління контекстом, безпека файлової системи, інтерактивні права доступу та зворотний зв'язок (feedback loop).

Шаблон повністю відповідає принципам меморандуму **AI-HomeLab**:
1. **Безпека та Гігієна** — не використовуються геополітично ризиковані інструменти чи моделі.
2. **Локальність** — повна підтримка локального інференсу через **Ollama** (Gemma 3, Llama 3.3).
3. **Економія** — низькі системні вимоги, придатний для роботи від EcoFlow.

---

## 🏗️ АРХІТЕКТУРА HARNESS

Агент побудований за модульним принципом, де модель відповідає лише за генерацію думок та викликів інструментів, а зовнішній Harness забезпечує безпеку та виконання:

```
                  ┌───────────────────────────────────────────┐
                  │          🧑‍💻 Термінал / CLI REPL          │
                  └─────────────────────┬─────────────────────┘
                                        │
                                        ▼
                  ┌───────────────────────────────────────────┐
                  │             🔁 Agent Loop                 │
                  └───────┬───────────────────────────▲───────┘
                          │                           │
         [Виклик інструменту]                      [Результат]
                          │                           │
                          ▼                           │
                  ┌───────────────────────────────────┴───────┐
                  │          🛡️ Permission Engine             │
                  │   (rm -rf, sudo, force push checks)       │
                  └───────┬───────────────────────────────────┘
                          │
                   [Дозволено / Дозволено Користувачем]
                          │
                          ▼
                  ┌───────────────────────────────────────────┐
                  │          💾 Workspace Safe Box            │
                  │ (CWD containment, Read-Before-Write, Diff)│
                  └───────┬───────────────────────────────────┘
                          │
                          ▼
                  ┌───────────────────────────────────────────┐
                  │             🔧 Виконання                  │
                  │     (File system, Read, Write, Bash)      │
                  └───────────────────────────────────────────┘
```

### Основні модулі:
* **`cli.py`** — точка входу, керування параметрами та REPL-цикл терміналу.
* **`agent.py`** — ядро циклу Agent Loop, детерміноване стиснення історії (`compact_basic.py`) та візуалізація `diff` перед записом.
* **`model.py`** — абстракція провайдерів. Містить адаптер для **Ollama** (який транслює виклики у формат OpenAI/Ollama та назад у формат Anthropic Messages) та офіційний клієнт **Anthropic**.
* **`tools.py`** — реєстр та код вбудованих інструментів (`read_file`, `file_edit`, `bash` тощо).
* **`fs_safety.py`** — безпека файлової системи: обмеження роботи строго всередині робочої директорії (CWD), перевірка розмірів файлів, автоматичний бекап файлу перед зміною у директорію `.agent/history/`.
* **`permissions.py`** — двигун безпеки: сканує bash-команди на небезпечні шаблони (`rm -rf`, `sudo`, `git reset --hard`) та блокує або запитує підтвердження.

## 🚀 ШВИДКИЙ СТАРТ

Для роботи рекомендується використовувати менеджер пакетів `uv`.

### 1. Встановлення залежностей

```bash
# Клонуйте репозиторій (якщо ще не зробили)
cd templates/agent-code-cli
uv sync
```

### 2. Локальний запуск (Ollama)

Переконайтеся, що Ollama запущена локально (наприклад, з моделлю `gemma3:4b` або `llama3.3:8b`):

```bash
# Запуск інтерактивного REPL режиму з локальною моделлю
uv run agent-code --provider ollama --model gemma3:4b
```

### 3. Хмарний запуск (Claude API / DeepSeek API)

Для використання провайдера `anthropic` налаштуйте API ключі:

```bash
# Для Claude
export ANTHROPIC_API_KEY="sk-ant-..."
uv run agent-code --provider anthropic --model claude-3-5-sonnet-latest

# Або для DeepSeek API (через сумісний endpoint)
export ANTHROPIC_AUTH_TOKEN="sk-..."
export ANTHROPIC_BASE_URL="https://api.deepseek.com/anthropic"
uv run agent-code --provider anthropic --model deepseek-v4-flash
```

## 🛡️ ЗАХИСТ ТА БЕЗПЕКА В ДІЇ

1. **Ізоляція Шляхів**: Модель не може виконати запит на читання чи запис файлу за межами вашого CWD. Спроба передати `../../etc/passwd` завершиться помилкою безпеки.
2. **Read-Before-Edit**: Модель зобов'язана спочатку прочитати файл (`read_file` або `grep`), перш ніж вона зможе запропонувати редагування (`file_edit`). Це гарантує, що модель розуміє поточний стан файлу та запобігає генерації некоректних латок.
3. **Захист від конфліктів**: Якщо ви змінили файл вручну після того, як модель його прочитала, Harness виявить зміну `mtime` та вимагатиме від моделі повторного читання файлу перед редагуванням.
4. **Контроль Bash**: Будь-яка bash-команда перед запуском вимагає підтвердження в консолі від користувача (натисканням клавіш `Y/N`), а небезпечні оператори (наприклад, `sudo`) блокуються на рівні регулярних виразів.
5. **Авто-бекап**: Кожна зміна файлу автоматично записує його попередній стан у директорію `.agent/history/` з часовою міткою, тому ви ніколи не втратите свій код через помилку моделі.
